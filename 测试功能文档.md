# 公考小助手测试功能文档

## 1. 目标

本轮测试重点覆盖以下高风险区域：

- Electron 主进程 IPC 返回结构是否与前端页面一致
- 错题本、学习计划、每日打卡、记忆卡片、成就系统的数据读写是否正常
- 本地日期计算是否按用户本地时间工作，而不是误用 UTC
- 数据导入/导出、番茄钟记录等辅助功能是否还能正常运行

## 2. 本轮已修复的核心问题

- `better-sqlite3 + Drizzle` 返回 camelCase 字段，前端页面实际依赖 snake_case，导致多个页面拿到空值或结构错误
- 错题本接口返回了嵌套 `question` 对象，而页面需要扁平字段
- 错题“到期复习”判断方向写反，原逻辑会把未来复习项当成到期项
- 成就解锁条件比较符号写反，原逻辑会导致未达成时也被解锁
- 连续打卡与“今天”判断使用 UTC 日期，跨时区后会出现当天/连续天数错误
- 学习计划页热力图文案写“近 30 天”，实际只加载了 7 天数据
- `npm run lint` 指向不存在的 `eslint`，导致验证命令本身失效

## 3. 自动化测试

### 3.1 运行命令

```bash
npm run lint
npm run test
```

### 3.2 自动化测试覆盖点

- IPC 兼容层是否输出前端当前使用的 snake_case 数据结构
- 错题本记录是否正确扁平化题目信息
- 记忆卡片是否返回 `review_count` / `next_review` 等旧字段
- 连续打卡计算是否允许“今天未打卡但昨天开始连续”的场景
- 到期复习判断是否按真实时间比较
- 成就进度统计是否只统计专注番茄钟、是否按去重日期统计打卡
- 成就解锁是否仅解锁已达到阈值且尚未解锁的记录
- 学习计划排序是否把进行中、高优先级任务放在前面

## 4. 手工功能测试清单

### 4.1 错题本

1. 新增一道错题，填写题干、选项、正确答案、我的答案、笔记。
2. 返回列表页，确认题型、题干、答案、解析、笔记都能展开显示。
3. 点击“标记掌握”，确认状态更新。
4. 点击“复习”，确认下次复习时间被刷新。
5. 删除错题，确认列表移除。

### 4.2 每日打卡

1. 设置考试名称和日期。
2. 选择学习时长，填写今日总结并保存。
3. 重新进入页面，确认今日记录、连续打卡、累计打卡、倒计时仍正确显示。

### 4.3 学习计划

1. 新建学习计划，填写科目、截止日期、优先级、每日时长。
2. 切换计划状态，确认计划列表排序和状态展示正常。
3. 打开“记录学习”，录入学习时长、做题数、错题数。
4. 确认今日统计、近 30 天热力图、本周学习时长同步更新。

### 4.4 记忆卡片

1. 新建卡片并保存。
2. 切换“待复习/已掌握/全部”筛选。
3. 翻卡后点击“会了/不会”，确认复习次数、掌握状态、下次复习日期正常变化。

### 4.5 成就系统 / 鼓励页

1. 打开成就页，确认未达成项不会提前解锁。
2. 录入学习、番茄钟、错题、卡片数据后重新进入，确认进度条随数据变化。
3. 打开鼓励页，确认新解锁成就只在首次解锁时弹出。

### 4.6 思维导图

1. 新建导图并保存。
2. 重新加载该导图，确认标题、学科、节点数据恢复正常。
3. 删除导图，确认列表同步移除。

### 4.7 番茄钟

1. 启动一个工作番茄钟并完成。
2. 确认每日学习时长增加，同时番茄钟记录表新增一条 `work` 记录。
3. 检查历史统计是否更新。

### 4.8 数据导入导出

1. 执行数据导出，确认生成 JSON 文件。
2. 再执行数据导入，确认主要页面数据仍能正常读取。

## 5. 打包与交付

### 5.1 构建命令

```bash
npm run build:all
npm run electron:build
```

### 5.2 产物位置

- `release/公考小助手 Setup <版本号>.exe`
- `release/公考小助手 <版本号>.exe`

## 6. 本轮建议的最终验收顺序

1. 先执行 `npm run lint`
2. 再执行 `npm run test`
3. 再执行 `npm run build:all`
4. 最后执行 `npm run electron:build`

如果以上 4 步全部通过，再进行 GitHub 同步和最终发版。
